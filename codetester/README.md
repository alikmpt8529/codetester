# コーディング規約チェッカー仕様書

## 全体仕様
このプロジェクトをgitにアップし、このファイルが仕様であることを明記する

## 基本機能

### 1. コーディング規約ファイルのアップロード
- 主要コーディング規約を指定する.txtファイルをアップロードする
- アップロードできたらそのファイル名を表示

### 2. 二次コーディング規約ファイルのアップロード（任意）
- 別のコーディング規約を指定する.txtファイルがあればそれもアップロードする
- アップロードできたらそのファイル名を表示

### 3. Cソースファイルのアップロード
- .cファイルをアップロードする
- アップロードできたらそのファイル名を表示

### 4. プログラムチェック
チェックは以下の手順で行う：
1. 2でアップされたファイル（二次規約）をチェック
2. 1でアップされたファイル（主要規約）をチェック

### 5. 結果出力
- 規約に準拠していなければ、どの部分が規約のどの部分に違反しているかを.txtファイルで出力する
- 規約に準拠していればcorrectと返り値を与える

## 追加機能

### 課題生成機能
- ２つ目に記載されたコーディング規約は課題１から４があるものとし、c言語ファイルの生成要請があればその数のｃ言語ファイルの提出箇所を課題１から順番に生成します

### 修正版ダウンロード機能
- 規約に違反していれば違反箇所をコメントアウトで追加したc言語ファイルをダウンロードできるようにし、そのボタンを作成する

## 安定性向上のための追加仕様

### エラーハンドリング
1. **ファイル形式チェック**
   - アップロード時にファイル拡張子を検証（.txt、.cのみ許可）
   - ファイルサイズ制限（最大10MB）
   - 不正なファイル形式の場合はエラーメッセージを表示

2. **ファイル内容検証**
   - 空ファイルのチェック
   - 文字エンコーディングの検証（UTF-8推奨）
   - 不正な文字が含まれている場合の警告表示

3. **メモリ管理**
   - 大きなファイルの処理時のメモリ使用量監視
   - 処理タイムアウト設定（30秒）
